# SKALA 스터디 플래너

HTML, CSS, JavaScript 수업에서 만든 스터디 플래너에 마감일, 검색, 분류별 요약 기능을 추가한 최종 제출 과제

Github : https://github.com/hyunooss/planner

## 1. 구현한 기능 목록

- F1 목표 추가
  - 빈 값이나 공백만 입력한 목표는 추가하지 않고 오류 메시지를 표시
- F2 완료 토글 및 삭제
  - 학습 목록 부모 요소에 하나의 클릭 리스너만 두는 이벤트 위임을 사용
- F3 필터 및 진행률
  - 전체, 진행중, 완료 필터를 제공
  - 전체 목표 수, 완료 목표 수, 완료 비율을 표시
- F4 localStorage 저장
  - 목표를 추가하거나 수정, 삭제할 때 브라우저에 저장
  - 페이지를 새로고침해도 목표 목록을 유지
- F5 마감일
  - `input type="date"`로 목표 마감일 입력 가능
  - 완료하지 않은 목표의 마감일이 지났으면 붉은 배경과 `기한 지남` 문구로 강조
- F6 실시간 검색
  - 검색어가 목표 제목에 포함된 항목만 입력과 동시에 표시
  - 검색과 전체, 진행중, 완료 필터를 함께 사용 가능
- F7 분류별 요약
  - 완료하지 않은 HTML, CSS, JS 목표의 개수를 `reduce()`로 집계해 표시
- 오늘의 팁
  - `fetch`, `async/await`, `try/catch`로 `data/tips.json`을 load
- 접근성 및 반응형
  - 모든 입력에 `label` 또는 `aria-label`을 연결
  - 375px 화면에서 가로 스크롤 없이 사용할 수 있도록 구성
  - 키보드 focus와 모션 감소 설정을 지원
- p.184 도전 과제
  - 목표 제목을 더블클릭하거나 Enter/F2 키를 눌러 수정 가능
  - 마감일 임박순과 최신순 정렬 버튼을 제공
  - CSS 사용자 지정 속성을 교체하는 다크 모드를 지원하고 설정을 저장
  - 목표가 없을 때 안내 문구와 입력 예시를 표시
  - 목표를 추가하거나 삭제할 때 부드러운 전환 효과를 적용

## 2. 실행 방법

`fetch()`가 JSON 파일을 읽을 수 있도록 로컬 웹 서버로 실행합니다.

```bash
cd planner
python3 -m http.server 5500
```

브라우저에서 아래 주소를 엽니다.

```text
http://localhost:5500
or
127.0.0.1:5500
```

저장된 목표를 초기 상태로 되돌리고 싶다면 브라우저 개발자 도구 콘솔에서
아래 코드를 실행한 뒤 새로고침합니다.

```js
localStorage.removeItem("skala-planner");
location.reload();
```

## 3. 어려웠던 점과 해결

- 기존 목표 데이터에는 마감일 필드가 없을 수 있었습니다.
  - 데이터를 불러올 때 `normalizeGoals()`로 `due` 필드와 분류 값을 보완해
    이전 데이터도 오류 없이 사용할 수 있도록 해결했습니다.
- 검색과 상태 필터를 동시에 적용하면서 화면과 데이터가 달라질 수 있었습니다.
  - `visible()`에서 상태 필터와 검색 조건을 순서대로 적용하고, 모든 변경 후
    `render()`가 현재 상태 전체를 다시 그리도록 구성했습니다.
- 사용자 입력을 목록에 안전하게 표시해야 했습니다.
  - 사용자 입력을 `innerHTML`에 넣지 않고 생성한 DOM 요소의 `textContent`에
    대입했습니다.
- 작은 화면에서 폼과 목록 항목이 넘칠 수 있었습니다.
  - Grid와 Flexbox, 미디어 쿼리로 375px에서는 폼과 요약을 세로로 배치하고
    긴 목표 제목이 줄바꿈되도록 처리했습니다.
